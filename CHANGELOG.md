## [v0.7.0] - 2026-05-17

### Highlights

- Threshold Matrix XLSX sheet with M1-M5 threshold columns
- SLI tags with kebab-case validation for multi-dimensional classification
- NIST CSF category ordering: govern → identify → protect → detect → respond → recover
- Dashboard bullet charts grouped by category with section headers

### Added

- Threshold Matrix XLSX sheet showing SLIs with Category, Tags, Frameworks, and M1-M5 threshold columns
- `Tags` field on SLI for multi-dimensional classification orthogonal to category
- Tag validation: lowercase kebab-case, 1-32 characters, starts with letter
- `ValidateTag()`, `NormalizeTags()`, `GetNormalizedTags()` functions for tag handling
- Recommended tag constants: ai, shift-left, supply-chain, runtime-defense, vulnerability-management, incident-response
- `Category` struct with ID, Name, Description, and SLIOrder for custom ordering
- `Categories` array in Spec for custom category and SLI display order
- `CategorySortWeight()` returns NIST CSF canonical order with common variations
- `DefaultCategoryOrder()` returns govern, identify, protect, detect, respond, recover
- `GetCategoryOrder()`, `GetCategoryByID()`, `GetSLIOrderForCategory()` helper methods
- Dashboard bullet charts grouped by category with NIST CSF sort order
- Dashboard flexbox layout for bullet labels with auto-calculated width
- Dashboard progress chart height scales dynamically based on SLI count

### Changed

- Dashboard bullet charts grouped by category instead of methodology (RED, USE, etc.)
- Threshold Matrix sorted by category (NIST CSF order) then alphabetically by SLI name
- Dashboard SLI tables sorted by category (NIST CSF order)
- Dashboard progress chart sorted by category then alphabetically by SLI name
- Chart heights calculated explicitly from grid layout for proper sizing

### Fixed

- Progress chart uses same data retrieval logic as bullet charts for consistency
- Bullet chart labels no longer truncated for long SLI names

### Documentation

- SLI tags documentation in docs/schema/slos.md
- Threshold Matrix documentation in docs/cli/maturity-xlsx.md
- v0.7.0 release notes

## [v0.6.0] - 2026-05-10

### Highlights

- BREAKING: Three-part schema taxonomy: Model, State, Plan
- BREAKING: CLI restructured to match schema taxonomy
- Domain-centric examples directory structure
- New MaturityStateDocument type for state tracking
- Consistent schema naming: prism-maturity-{model,state,plan}.schema.json

### Breaking

- CLI: `prism maturity report` → `prism maturity model report`
- CLI: `prism maturity xlsx` → `prism maturity model xlsx`
- CLI: `prism dashboard` → `prism maturity plan dashboard`
- CLI: `prism report` → `prism maturity plan report`
- Schema files renamed: `maturity.schema.json` → `prism-maturity-model.schema.json`
- Schema files renamed: `prism.schema.json` → `prism-maturity-plan.schema.json`
- Examples directory restructured to domain-centric layout
- Old schema embedding functions deprecated in favor of new names

### Added

- `prism maturity model` subcommand: report, xlsx, dashboard, validate
- `prism maturity state` subcommand: validate, show
- `prism maturity plan` subcommand: dashboard, report
- `prism maturity model dashboard` with --state flag for state document integration
- `prism maturity model validate` for model document validation
- `prism maturity state validate` with --model flag for cross-validation
- `prism maturity state show` with text/json output formats
- `prism maturity model lint` command for identifying dashboard display issues and threshold gaps
- Dashboard qualitative threshold display showing 'Tracked' for `exists` operator
- Dashboard qualitative state in SLI subtitles (e.g., 'Measured (M1)', 'Tracked (M1)')
- Dashboard 'Not Tracked' display for qualitative SLIs without state data
- Dashboard '-' display for maturity levels without defined thresholds
- Security methodology groupings: Prevention, Detection, Response
- AI security methodology groupings: AI Prevention, AI Detection, AI Response
- New `prism-maturity-state.schema.json` for state tracking documents
- `MaturityStateDocument` type for PRISM Maturity State documents
- `MaturityStateMetadata` type with maturityModelRef field
- `MaturityModelSchemaJSON()`, `MaturityStateSchemaJSON()`, `MaturityPlanSchemaJSON()` functions
- Schema constants: `MaturityModelSchemaID`, `MaturityStateSchemaID`, `MaturityPlanSchemaID`
- Domain directories: `examples/operations/`, `examples/security/`, `examples/organization/`, `examples/quality/`
- `schema/generate_state.go` for state schema generation

### Changed

- Examples organized by domain instead of by document type
- All example files updated with new schema references
- Test file paths updated for new examples structure

### Deprecated

- `PRISMSchemaJSON()` - use `MaturityPlanSchemaJSON()` instead
- `PRISMSchemaMap()` - use `MaturityPlanSchemaMap()` instead
- `SchemaID()` - use `MaturityPlanSchemaID` instead

### Fixed

- Dashboard threshold display for models without sliType (flat bullet list now includes thresholds)

### Documentation

- Updated README with three-schema taxonomy
- Updated examples/README.md with domain-centric structure
- Migration guide for v0.5.0 → v0.6.0

## [v0.5.0] - 2026-05-10

### Highlights

- BREAKING: Separated PRISM Maturity Model (definition) from PRISM Maturity State (tracking)
- New temporal SLO window support: 7d, 30d, 90d, quarterly, annual
- Hybrid qualitative/quantitative criteria with measurement type classification
- Dashboard dual-read support for backward compatibility during migration

### Breaking

- Removed `assessments` field from maturity model files - state tracking now uses separate PRISM Maturity State documents
- `Spec.Assessments` field is deprecated and will be removed in a future release
- `DomainAssessment` type is deprecated in favor of the new PRISM Maturity State format
- Example maturity model files (`examples/maturity-models/*/model.json`) no longer contain assessment data

### Added

- `QualitativeState` type for qualitative progression tracking (none, adhoc, tracked, measured, alerting, optimized)
- `MeasurementType` constants: quantitative, qualitative, hybrid
- `SLI.MeasurementType` and `SLI.QualitativeStates` fields for hybrid metric support
- `SLI.GetMeasurementType()`, `GetQualitativeStates()`, `IsQuantitative()`, `IsQualitative()`, `IsHybrid()` helper methods
- `Generator.WithStateDocument()` method for dashboard dual-read support
- New PRISM Maturity State examples in `examples/maturity-state/` directory
- Temporal window support in state documents: 7d, 30d, 90d, quarterly, annual SLO tracking

### Changed

- XLSX generator handles nil assessments gracefully (no panic when assessments not present)
- Dashboard generator supports reading state from separate PRISM Maturity State document

### Documentation

- `docs/design/REFACTOR_MATURITY_STATE.md` with architecture for model/state separation
- Updated `examples/README.md` with new directory structure and naming conventions
- Migration guide for upgrading from v0.4.0 to v0.5.0

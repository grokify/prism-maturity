# v0.10.0 Release Notes

**Release Date:** 2026-05-22

## Highlights

- **PDF Export for Layer Views** - Export layer-based and category-based metric views to PDF
- **Aggregation Method CLI Flag** - Choose between min (conservative) or avg (average) aggregation
- **Capability Stack Path Flag** - Specify capability stack for layer-based maturity views

## Added

### PDF Export for Metrics Views

Layer and category metric pages now include PDF export functionality:

**Per-Group Export:**

- Click the PDF button on any layer/category header
- Generates a single-page PDF with metric table

**Export All:**

- Click "Export All Layers" or "Export All Categories"
- Generates a multi-page PDF with:
  - Summary page with all groups and average levels
  - Detailed page per group with full metric tables

**PDF Contents:**

| Column | Description |
|--------|-------------|
| Metric | SLI name |
| Current | Current maturity level (M1.0 - M5.0) |
| M1-M5 | Threshold values for each level |

### CLI Flags

Two new flags for `prism site generate`:

```bash
prism site generate \
  --stack=./security/ \
  --capstack=./stacks/security/stack.json \
  --aggregation=min \
  --output=./dist
```

**`--capstack`**

Path to capability stack file for layer-based maturity views.

**`--aggregation`**

Aggregation method for layer-level maturity calculation:

| Value | Description |
|-------|-------------|
| `min` | Minimum SLI level (default, conservative) |
| `avg` | Average SLI level (balanced view) |

## Changed

### Config Struct

The `site.Config` struct now includes:

```go
type Config struct {
    // ... existing fields ...
    CapStackPath      string  // Path to capability stack for layer views
    AggregationMethod string  // "min" or "avg"
}
```

### MaturityAggregator

The aggregator now respects the configured aggregation method:

```go
// Uses config.AggregationMethod to determine aggregation
switch g.config.AggregationMethod {
case "avg":
    aggMethod = dashboard.AggregationAvg
default:
    aggMethod = dashboard.AggregationMin
}
```

## Usage Examples

### Generate Site with Layer Views

```bash
prism site generate \
  --stack=./examples/security/ \
  --aggregation=min \
  --output=/tmp/prism-test
```

### Test PDF Export

1. Generate the site
2. Open `metrics-by-layer-{stack}.html`
3. Click PDF button on any layer header
4. Click "Export All Layers" for full report

### Verify CLI Flags

```bash
# Show help with new flags
prism site generate --help | grep -E "capstack|aggregation"

# Test invalid aggregation (should error)
prism site generate --stack=./test --aggregation=invalid 2>&1
```

## Migration Guide

No breaking changes. Existing sites continue to work with default `min` aggregation.

To enable average aggregation:

```bash
prism site generate --stack=./stacks/ --aggregation=avg --output=./dist
```
